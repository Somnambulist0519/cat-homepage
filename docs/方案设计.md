# 猫的主页 · 方案设计文档（PRD）

> **【置顶规则 · 贯穿全程】**
> 在「确保功能正常运行」的前提下，**保持根目录与项目结构简洁**：
> - 用户每提出一次新要求或修改建议，**不要无脑往文件夹里堆新文件**；
> - 凡是被替代、过期、仅模板遗留且无用的内容，**立即删减**；
> - 同一件事只保留一个权威来源（如包管理器锁文件只留一个、配置只留一个入口）；
> - 新功能优先「复用 / 改写现有文件」，而非新建平行文件。
> 本规则是验收标准之一：每次交付都要让目录比上一版更干净。

---

## 1. 背景与目标

### 1.1 缘起
用户要一个类 `blog.notionedu.com` 的个人博客，定位为**可 Fork 分享的模板**：别人拿到后，能像搭积木一样选主题、功能块、布局、视觉细节，构建出自己喜欢的页面；而**读者前台只看到开发者设定好的成品，完全看不到搭建过程**。

### 1.2 已完成的地基（截至本文）
- 技术栈：**Astro v7 + Tailwind v4 + Astro Paper v6.1.0** 模板。
- 已落地：中文站（`zh-CN`）、苹果极简风 → 克制版**玻璃拟态**、全屏差异化宽度、可自定义背景图、文章页**双栏（作者卡 + 目录 TOC）**、本地预览可用、构建通过。
- 关键文件：`src/site.config.ts`（个人信息/作品/联系/COS）、`src/styles/theme.css` + `global.css`（设计令牌/玻璃工具类）、`src/layouts/Layout.astro`（背景图注入）、`src/components/VideoEmbed.astro`（B站/YouTube/COS 视频）、`src/components/AuthorCard.astro` + `TableOfContents.astro`。

### 1.3 目标
一个**「主题货架 + 搭建向导 + 零代码后台」**三位一体的个人站模板：
1. **主题货架**：8–10 套皮肤预设，覆盖不同审美与人群。
2. **搭建向导 `/setup`**：fork 后本地跑命令打开，左选右预览，保存即固化配置；**生产包不含向导**。
3. **内容后台 `/admin`**：Sveltia CMS，网页写文章/传图传视频，零后端。
4. **媒体内嵌**：正文贴链接自动变播放器（B站/YouTube/COS）。
5. 部署：腾讯云 COS + GitHub Actions 自动发布。

## 2. 总体架构

### 2.1 技术栈
| 层 | 选型 | 说明 |
|---|---|---|
| 框架 | Astro v7（静态输出） | Jamstack，构建期生成纯静态 HTML |
| 样式 | Tailwind v4（`@utility` + CSS 变量令牌） | 设计令牌走 CSS 变量，主题切换即换变量 |
| 基底模板 | Astro Paper v6.1.0 | 已裁剪为符合需求的最小集 |
| 后台 | Sveltia CMS（Git-based） | GitHub OAuth PKCE 登录，免认证服务器 |
| 部署 | 腾讯云 COS + GitHub Actions | 推 main 自动构建并上传 |
| 搜索 | Pagefind | 构建期索引，纯静态 |

### 2.2 目录结构与职责（目标态，按简洁规则收敛）
```
猫的主页/
├─ src/
│  ├─ site.config.ts        # 个人信息（名字/简介/作品/联系/COS）— 人写
│  ├─ data/
│  │  ├─ themes.ts          # 8–10 套主题预设（token + 视觉 flag）
│  │  └─ theme.config.ts    # 激活配置（向导写入；含功能块开关/布局/视觉覆盖）
│  ├─ styles/
│  │  ├─ theme.css          # 基础令牌 + 玻璃/暗色变量（被主题覆盖）
│  │  ├─ global.css         # 工具类 glass / app-wide / app-read
│  │  └─ typography.css
│  ├─ layouts/Layout.astro  # 注入激活主题 tokens（CSS 变量）+ 背景图
│  ├─ components/           # Header/AuthorCard/TableOfContents/VideoEmbed/...
│  ├─ pages/
│  │  ├─ index.astro        # 首页 Landing
│  │  ├─ posts/[...slug]/    # 文章（双栏）
│  │  ├─ works.astro / about.astro / search.astro / tags/ / archives/
│  │  ├─ setup.astro        # 搭建向导（仅 dev 暴露；生产排除）
│  │  └─ api/setup.ts       # dev-only 写盘接口（保存配置）
│  └─ content/{posts,works} # Markdown 内容（Sveltia 管理）
├─ public/                  # 静态资源（头像/背景图/作品封面/OG）
├─ docs/方案设计.md          # 本文件（蓝图）
├─ astro-paper.config.ts    # 引擎配置（功能总开关 features.*）
├─ astro.config.ts
├─ .github/workflows/       # COS 部署
└─ README.md                # 模板使用说明（fork 三步）
```
**收敛点**：原有的 `CHANGELOG.md`/`cz.yaml`/`compose.yaml`/`AstroPaper-lighthouse-score.svg` 等模板遗留物，在 Phase 4 清理；锁文件只保留项目实际使用的那一个。

### 2.3 数据流（主题如何生效）
```
themes.ts(预设)  ──┐
                   ├─→ theme.config.ts(激活项) ─→ Layout.astro 注入 CSS 变量 ─→ 全站样式
site.config.ts ───┘   (个人信息/背景图)
```
- 构建期：`Layout.astro` 读取 `theme.config.ts` 的激活主题，把对应 `themes.ts` 的 token 以 `<style set:html>` 注入 `:root` / `[data-theme=dark]`。
- 预览期（向导）：`/setup` 通过 `?preview=<themeId>` 让 `Layout` 临时套用某预设，**不落盘**，仅本地可见。

## 3. 主题货架清单（8–10 套预设）

每套预设 = `{ id, name, 描述, 适用人群, tokens{bg/surface/text/accent/glass*}, 视觉flag{glass, gradientBg, radius, font} }`。

| # | id | 名称 | 风格要点 | 适用人群 |
|---|---|---|---|---|
| 1 | `apple` | 苹果极简 | 近白底、低饱和灰、蓝点缀、无玻璃、克制阴影 | 通用 / 产品人 |
| 2 | `glass` | 玻璃月光 | 柔和渐变底 + 半透明磨砂卡 + 发丝描边（**当前默认**） | 设计 / 作品集 |
| 3 | `dark-neon` | 暗夜霓虹 | 近黑底、青/品红霓虹点缀、玻璃带微辉光 | 极客 / 二次元 |
| 4 | `editorial` | 报纸杂志 | 衬线标题、暖纸色、经典排版、极简卡 | 写作者 / 专栏 |
| 5 | `bento` | Bento 网格 | 圆角卡片拼贴、活泼强调色、网格感强 | 多面手 / 作品墙 |
| 6 | `mint` | 薄荷清新 | 浅薄荷/teal 粉彩、柔和友好 | 生活 / 女性向 |
| 7 | `ink` | 墨痕国风 | 宣纸米白、墨黑 + 朱砂点缀、衬线 | 文化 / 国风 |
| 8 | `terminal` | 极客终端 | 深底、等宽字体、绿字终端感、直角 | 开发者 / 工具控 |
| 9 | `sunset` | 日落暖阳 | 橙粉渐变、温暖治愈 | 摄影 / 旅行 |
| 10 | `morandi` | 莫兰迪 | 低饱和大地色、高级灰、克制 | 艺术 / 极简控 |

> 视觉 flag 说明：`glass`(是否启用磨砂卡)、`gradientBg`(是否用渐变占位背景)、`radius`(圆角 px)、`font`(标题/正文字体族)。预设给出默认值，向导里可单独覆盖。

## 4. 功能块系统

### 4.1 功能块清单（开关 = `theme.config.features.*`）
| 功能块 | key | 默认 | 说明 |
|---|---|---|---|
| 首页 Hero | `showHero` | ✓ | 名字 + 标语 + 简介 |
| 作品墙 | `showWorks` | ✓ | 首页作品卡（也读 `src/content/works`） |
| 视频区 | `showVideo` | ✓ | 首页 B站/YouTube/COS 嵌入位 |
| 联系/订阅 | `showContact` | ✓ | 表单（endpoint 预留） |
| 站内搜索 | `showSearch` | ✓ | Pagefind |
| 文章目录 | `showToc` | ✓ | 文章页 TOC |
| 作者卡 | `showAuthorCard` | ✓ | 文章页右侧 |
| 标签 | `showTags` | ✓ | 文章/列表标签 |
| 归档 | `showArchives` | ✓ | 时间线 |
| RSS | `showRSS` | ✓ | 订阅 |

### 4.2 款式（基础几款，向导可选）
- **卡片款式**：`glass`（磨砂）/ `flat`（纯色无阴影）/ `outline`（仅描边）。
- **按钮款式**：`pill`（全圆角）/ `rounded`（小圆角）/ `square`（直角）。
- **列表款式**：`card`（卡片流）/ `row`（紧凑行）。

## 5. 搭建向导 `/setup` 设计

### 5.1 定位与隔离
- **仅本地开发可用**：`astro dev` 时访问 `/setup`；生产构建（preview/build）**不暴露**向导、不留任何向导代码到 `dist`。
- 读者前台：**永远看不到**搭建过程，只看到固化后的成品。

### 5.2 界面（左选 · 右预览）
```
┌─────────────────────────────┬──────────────────────────┐
│ 左：配置面板（纯前端）        │ 右：实时预览（iframe）      │
│  • 主题货架（10 套缩略卡）    │  src 站 ?preview=<themeId> │
│  • 功能块开关（10 个 toggle） │  + ?features=... 覆盖      │
│  • 布局宽度（首页/文章）       │  随左栏操作即时刷新         │
│  • 视觉细节（圆角/卡片/按钮）  │                            │
│  • 强调色 / 字体             │                            │
│  ┌───────────────────────┐  │                            │
│  │ [保存配置] [重置默认]   │  │                            │
│  └───────────────────────┘  │                            │
└─────────────────────────────┴──────────────────────────┘
```

### 5.3 保存机制（缝合 whono 思路：开发可写 / 生产只读）
- 左栏为纯前端；点「保存」→ `POST /api/setup` → **dev-only** API 路由把配置写成 `src/data/theme.config.ts`（生成带类型的 TS，含注释）。
- 生产环境该 API 返回 404，且 `/setup`、`/api/setup` 路由在构建时被条件排除（`import.meta.env.PROD` 守卫）。
- 配置一旦落盘，下次 `astro build` 即固化进成品。

### 5.4 对标缝合点
- **OpenPage**：JSON 配置 + 多主题 + 实时预览 → 用 `theme.config.ts` + iframe 预览。
- **whono Theme Console**：开发态可写、生产只读 → 用 `import.meta.env.PROD` 守卫。
- **EmDash**：setup wizard 流 → 左选右预览的交互骨架。

## 6. 内容管理（Sveltia CMS `/admin`）

- Git-based 后台，GitHub OAuth（PKCE）登录，**无需自建认证服务器**。
- Collections 映射：`posts` → `src/content/posts/*.md`；`works` → `src/content/works/*.md`；媒体 → `public/`。
- 让 fork 用户**不改代码**即可发文、传图、传视频。
- 与 `/setup`（外观）职责分离：外观用向导，内容用后台。

---

## 7. 媒体嵌入规范

- 正文（Markdown）贴一行链接 → 自动转播放器：
  - B站：`player.bilibili.com`（`b23.tv`/`bilibili.com/video` 识别）。
  - YouTube：`youtube.com/watch` / `youtu.be` → embed。
  - 腾讯云 COS：直接 `<video>` 直链。
- **不支持**：抖音（无公开 embed）、YouTube 对无科学上网访客不可见（已在需求中确认，仅作可选）。
- 图文混排：Markdown 原生 `![alt](url)` + 居中/宽度工具类即可。

## 8. 对标差异（GitHub 调研结论）

| 项目 | 技术栈 | 向导时机 | 与我们差异 |
|---|---|---|---|
| OpenPage | Astro + JSON 配置 | 构建前配置 | 主题/功能块更丰富，我们有向导 UI |
| whono Theme Console | Astro | 开发可写/生产只读 | 思路一致，我们加实时预览 |
| EmDash | Astro | setup wizard | 我们向导更偏向"外观货架" |
| Firefly | Astro | 多布局+色相 | 我们主题更多、更贴合中文审美 |
| GrapesJS/Webstudio | 重 + 后端 | 可视化拖拽 | 太重，不符"零后端纯静态" |

**结论**：无完全对标项目。**缝合三者之长**——OpenPage 的配置化 + whono 的生产隔离 + EmDash 的向导流——实现本地 `/setup` 向导。

## 9. 实施阶段（Roadmap，可分步验收）

| 阶段 | 内容 | 验收 |
|---|---|---|
| **Phase 1** | 主题配置系统：`themes.ts`(10 预设) + `theme.config.ts`(激活/类型/开关) + `Layout` 注入 token | `astro build` 通过；切 `theme` 字段即换肤 |
| **Phase 2** | 搭建向导 `/setup` + dev-only `/api/setup` + iframe 预览 | 本地选主题/开关即时预览；保存落盘；prod 不可见 |
| **Phase 3** | Sveltia CMS `/admin` | 网页登录 GitHub 后发文/传媒体 |
| **Phase 4** | 媒体内嵌完善 + 模板 README + **根目录清理**（删模板残留/多余锁文件） | 正文链接转播放器；README fork 三步；目录更干净 |

每阶段结束都跑 `astro build` 确认不回归，并落实「简洁规则」。

## 10. 风险与约定

- **主题一致性**：所有视觉差异必须收敛到 CSS 变量，禁止组件内硬编码颜色，保证切主题零副作用。
- **生产隔离**：`/setup`、`/api/setup` 必须 `import.meta.env.PROD` 守卫，防止向导代码泄露到成品。
- **简洁规则（置顶）**：每次变更都要删减冗余、收敛来源、复用而非新建。
- **中文优先**：界面文案、README、注释以中文为主。
- **不联网构建**：禁用依赖外部字体/接口的动态能力（沿用已验证的静态 OG、系统字体方案）。
